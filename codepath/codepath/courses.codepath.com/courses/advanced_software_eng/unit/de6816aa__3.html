Week 3 - Review

This week we will be reviewing the concepts we've covered so far.

Goals

  • Practice time and space complexity analysis
  • Get comfortable with problem identification
  • Review linked lists, hash table, stacks, queues, heaps problems

Before our weekly session

Due before Monday, June 17th at 11:59pm PDT

Check out the warmups session tab to review the required tasks before our weekly session. This will include reviewing basic concepts related to this week's topic and doing a few easy/medium coding problems. Plan to spend between 1 and 5 hours on these exercises. if you haven’t tried Pramp yet, now might be a great time to test our your skills in a mock interview. If you sign up with http://pramp.com/promo/codepath, you’ll get unlimited interviews (I think it’s normally limited to 3). If you already have an account, you might need to create a new account to get the unlimited interviews. The interviewers can be hit or miss, but even with a less experienced interviewer, it’s great practice for your technical communication, and to see if you can quickly apply the UMPIRE method in a timed setting.

In class exercise #1: Interview question walkthrough

Question: Smallest Range

Solution (in Python): Solution

Before the live demo of this question, spend around 25 minutes doing the following for this problem:

  1. Understand the problem. Besides the given example(s) try to come up with some of your own examples for the problem.

  2. Match. Does this problem match any other patterns you've seen before?

  3. Prototype and pseudocode. Using one of the examples you came up with in Step 1 walk through a high level implementation. Make sure you're not missing critical edge cases.

  4. Reflect. What are the time and space complexities of your solution?

Review - Session 2

Schedule

  1. Advice/ tips on code cleanliness
  2. First mock interview
  3. Feedback & recap of first mock interview
  4. Second mock interview
  5. Feedback & recap of second mock interview
  6. Recap of mock interview questions

Instructions for in class exercises

We have 40-45 minutes for two questions:

  1. Evaluate Reverse Polish Notation
    • Aim to have a working solution with a set of test cases finished for this problem before moving on to the next one.
  2. Trapping Rain Water
    • This is a more challenging problem, aim to have pseudocode done by the end of this segment.

For each question:

  1. Create a new codeinterview session document to work on.

  2. Talk through problems - Select one person to code the solution and rotate for the next question. The other person should work with the coder to come up with a solution. This isn't just about getting a correct solution but also effective communication.

    • Come up with a few new example input/ outputs
    • Talk about the edge cases
    • Talk about different approaches
    • Pseudocode
    • Code the solution (talk throughout this segment!)
    • Test your solution

  3. Review solution - After completing a problem, scroll down to the bottom of this tab to find the solution links and discuss/compare your solution with the solution given. Is the one you've developed less efficient or less concise? Repeat this and work through the exercises below.

As you go through the questions below, be sure to rotate the person who is leading each question.
