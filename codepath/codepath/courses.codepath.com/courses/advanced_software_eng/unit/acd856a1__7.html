Pair Programming

For today's in class exercise, we are going to try pair programming for our challenge questions.

When doing pair programming, each person has one of two roles: Driver or Navigator.

The Navigator is in charge of thinking about how to approach the problem, and catching bugs while the Driver is typing out code.

The driver is in charge of writing the actual code.

  1. Start a new document in repl.it (or whatever your favorite code collaboration editor is), and invite your group members.
  2. Start off by having each person write one test case as part of the Understand step in UMPIRE.
  3. Talk through different approaches of how to solve the problem and agree on one approach to code up.
  4. The person who originally came up with the approach of how the solve the problem should be the Navigator while the other person should act as the Driver. (If the solution was agreed upon together, randomly assign the roles).
  5. The Navigator should explain their thought process so that the Driver has enough information and context to code up the solution.
  6. Once you've finished one problem, switch off with the Navigator / Driver roles and start on another problem.

Lecture Slides

Debugging

In the session we will work on our debugging skills.

One of the most stressful parts of an interview is trying to debug your own code under time pressure. You’ve implemented a solution, but now the interviewer gives an example of when the implementation doesn’t work correctly and you’re tasked with fixing the code so that it’ll work properly.

We have provided some buggy code snippets along with some test cases. The goal for your group will be to fix the code snippets so all the test cases passes.

Once you and your group have decided whether to work on the problem in Java or Python:

  1. Copy the buggy code from HackMD to a codepad
  2. Make sure the code runs and gives outputs
  3. Read through and understand the problem
  4. Understand the ‘almost’-solution and trace through the code with one of the test cases we provide
  5. Try to fix the code so all tests pass (Try to fix it without print statements first. Imagine if this was like a whiteboarding interview.)

The list of challenge questions along with their code can be found here: https://docs.google.com/document/d/1Sz0smiZFKh7bEAYwNuMHwP213sCPi5nAvcd76AEg1YY/
