Week 6 - Strings and Arrays

This week we will be covering the fundamentals of strings and arrays and common ways they appear in interviews.

Goals

  • Comfort with traversing a 2x2 Matrix
  • Recognize and know when to utilize a binary search
  • Knowing sorting algorithms and their runtimes

Before our weekly session

Due before Monday, July 8th at 11:59pm PDT

Check out the warmups session tab to review the required tasks before our weekly session. This will include reviewing basic concepts related to this week's topic and doing a few easy/medium coding problems. Plan to spend between 1 and 5 hours on these exercises. if you haven’t tried Pramp yet, now might be a great time to test our your skills in a mock interview. If you sign up with http://pramp.com/promo/codepath, you’ll get unlimited interviews (I think it’s normally limited to 3). If you already have an account, you might need to create a new account to get the unlimited interviews. The interviewers can be hit or miss, but even with a less experienced interviewer, it’s great practice for your technical communication, and to see if you can quickly apply the UMPIRE method in a timed setting.

After the session

Due before Monday, July 15th at 11:59pm PDT

Check out the assignment tab to review the required tasks after attending our weekly session. There will be some post-session questions that you can work on throughout the week to help solidify your understanding. Some problems may be optional, but it is highly recommended that you do all of them. Working on these questions will help you succeed in the final HackerRank assessment for this unit. The HackerRank assessment will be a mix of multiple choice and coding challenges. Plan to spend 90 minutes on this assessment.

Strings and Arrays - Session 1

Lecture slides

Exercise #1 (20 minutes)

Question: Given a string s , find the length of the longest substring t that contains at most 2 distinct characters.

Example 1:

  • Input: "eceba"
  • Output: 3
  • Explanation: t is "ece" which its length is 3.

Example 2:

  • Input: "ccaabbb"
  • Output: 5
  • Explanation: t is "aabbb" which its length is 5.

Instructions

  1. Understand the problem. Besides the given example(s) try to come up with some of your own examples for the problem.

  2. Match. Does this problem match any other patterns you've seen before?

  3. Prototype and pseudocode. Using one of the examples you came up with in Step 1 walk through a high level implementation. Make sure you're not missing critical edge cases.

Exercise #2 (20 minutes per question)

Create a new codepad document to work on.

For the following questions:

  1. Understand the problem. Besides the given example(s) try to come up with some of your own examples for the problem.

  2. Match. Does this problem match any other patterns you've seen before?

  3. Prototype and pseudocode. Using one of the examples you came up with in Step 1 walk through a high level implementation. Make sure you're not missing critical edge cases.

If you have extra time: code up your solutions and run test cases through them!

Keep in mind that during interviews coding questions are usually allotted 30-40 minutes. Given that time constraint, you should aim to spend around 15-20 minutes to get to the pseudocode for each problem:

  1. Set matrix zeros
  2. Group anagrams
  3. Pow x n
  4. Multiply Strings

Strings and Arrays - Session 2

Lecture Slides

Instructions for in class mock interviews

Refer to this mock interview guide

Introductions

Introduce yourselves to each other! What school do you go to? Where are you from?

Mock interview prep (15 minutes)

Designate one person as person A, the other as person B.

Person A will review this problem and then ask person B this problem:

Max chunks to make sorted

Person B will review this problem and ask person A this question

Basic calculator

Spend 15 minutes prepping out the problem you were assigned. Below is a breakdown of how you should spend the fifteen minutes:

  1. Read and understand problem
  2. Think through possible test cases and edge cases
  3. Read through the solutions to the problem
  4. Create a new codepad document and save the URL to be shared during the mock interviews.

(If time permits) Come up with 3 helpful hints you could give when you’re conducting the interview to help the other person become ‘unstuck’

Mock interviewing (45 minutes each round)

There will be 2 rounds of mock interviewing. Each round will be 45 minutes long and there should be a feedback period following each interview.

  1. Person A will begin the first round of interviews as the interviewer

    • Interviewer introduces the problem, interviewees clarify the question
    • Interviewees come up with example input/ output and write down pseudocode
    • Actual coding
    • End with a quick time/ space complexity analysis even if the coding is not complete.

  2. After the interview is over, provide feedback for how the session went.

    • Example feedback for the interviewees:

      • What did the interviewee do well?
      • Did the interviewees come up with an example input/ output before planning out the approach?
      • Did the interviewees plan sufficiently before coding?
      • Were the interviewees communicative throughout?

    • Example feedback the interviewer:

      • What did the interviewer do well?
      • Did the interviewer clearly state the question?
      • Did the interviewer help along the way?

  3. Repeat steps 1 & 2 with person B as the interviewer.

Extra time?

Try going working through this question with your partner: https://leetcode.com/problems/maximum-gap/description/
