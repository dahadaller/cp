Recap and Design Questions - Session 1

Lecture Slides

In class exercise: Design a parking lot

Question: I’ve opened a mall and I want to build a parking lot for my new mall. How would you model this parking lot?

For this question, discuss with your partner...

  • What are the core classes involved in designing a parking lot?
  • What would be methods associated with each of those classes?
  • What data structures would you need to utilize to store the proper data?
  • What questions would you ask your interviewer to narrow down the scope of the question?

In class exercises, part 2 (50 minutes)

Aim to spend around 15 - 20 minutes on each problem and go through the object oriented design steps we covered in class:

  1. Narrow down the scope of the problem
  2. Define important features
  3. Define core classes
  4. Reflect on how the core classes relate to one another
  5. Define class attributes
  6. Define methods for each class

Question #1: Design circular queue

Question #2: Design a hash table.

Design and implement a hash table that uses chaining (linked lists to handle collisions). What are the main methods you'd need to support for this hash table?

Question #3: Design an in memory file system.

Create the classes and methods within those classes for this in memory file system.

Extra mock interview practice

Instructions for in class mock interviews

Refer to this mock interview guide

Mock interview prep (10 minutes)

Designate one person as person A, the other as person B.

Person A will review this problem and then ask person B this problem: Insert Delete GetRandom O(1)

Person B will review this problem and ask person A this question: LRU Cache

Spend 10 minutes prepping out the problem you were assigned. Below is a breakdown of how you should spend the fifteen minutes:

  1. Read and understand problem
  2. Think through possible test cases and edge cases
  3. Read through the solutions to the problem
  4. Create a new codepad document and save the URL to be shared during the mock interviews.

(If time permits) Come up with 3 helpful hints you could give when you’re conducting the interview to help the other person become ‘unstuck’

Mock interviewing (30 minutes each round)

There will be 2 rounds of mock interviewing. Each round will be 35 minutes long and there should be a feedback period following each interview.

  1. Person A will begin the first round of interviews as the interviewer

    • Interviewer introduces the problem, interviewees clarify the question
    • Interviewees come up with example input/ output and write down pseudocode
    • Actual coding
    • End with a quick time/ space complexity analysis even if the coding is not complete.

  2. After the interview is over, provide feedback for how the session went.

    • Example feedback for the interviewees:

      • What did the interviewee do well?
      • Did the interviewees come up with an example input/ output before planning out the approach?
      • Did the interviewees plan sufficiently before coding?
      • Were the interviewees communicative throughout?

    • Example feedback the interviewer:

      • What did the interviewer do well?
      • Did the interviewer clearly state the question?
      • Did the interviewer help along the way?

  3. Repeat steps 1 & 2 with person B as the interviewer.

  • Flatten nested list iterator
